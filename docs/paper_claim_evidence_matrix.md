## 关于 C4/C5 的关键限制

`candidate_pairs_after_topk == exact_gain_calls` 和 `exact_gain_positive_count < exact_gain_calls` 是 default legacy path 的直接漏斗事实；它支持 retained set 需要 exact validation。但仓库没有 `proxy_gain`/`proxy_ratio`/proxy rank 与 exact rank 的逐 pair联表，因此 C4/C5 不能扩写成 proxy ranking recall、correlation、regret或 final counterfactual claim。完整状态见 `docs/paper_proxy_exact_audit_status.md`。
