# PVLDB 论文证据清单

审计基于工作树 HEAD `71723127069fffcc9f136420a4cf88fe34919e34`；各实验自身 commit 以原文为准。当前工作树并不等于所有历史结果的生成版本。候选证据严格分类为：A = default EA-proxy versus exact gain；B = experimental CandidateIndex ranking；C = online sequence/final partition。仓库中没有完整 Category A 数据。

## 环境与不可混合边界

- **Ryzen Stage 0--5 环境**：已知部分使用 AMD Ryzen 7 8840HX、24 threads、CPU scoring；Stage 0 明确 commit `62365ca...`，其他 stage 多数未记录 commit。
- **PVLDB 历史环境**：`../paper/pvldb/main.tex` 描述 dual Intel Xeon Silver 4316、Tesla P100。`results/*.csv` 多数未记录 machine/commit/threads；不得与 Ryzen runtime 合并。
- **MAGS-DM 外部日志**：`results/mags_logs.docx` 显示 40 threads、`/data/zby/datasets/...` 和 2026-07-20 日志，但 docx metadata 是 python-docx 默认时间，machine/commit 未知。
- **objective 边界**：历史 `cost_ratio`、Stage 0 legacy merge objective、以及 Stage 1+ `mags-compatible` exact objective不是同一个算法配置。即使 CSV 同时打印多个 ratio，也不能据此宣称 candidate decisions 使用相同 objective。

## 文件覆盖说明

上述 evidence set 覆盖 `docs/` 下所有现有顶层 Markdown/CSV/SHA 文件；`stage0_logs/`、`stage0_profile_runs/`、`stage0_overhead_runs/` 按 E00--E02 作为 raw/supporting artifacts 整组登记。测试输出 `tests/outputs/**` 是 E00/E11 的小图 payload fixture；`.pyc` 是派生缓存，标记为 `DUPLICATE_RESULT`，不作证据。所有 test datasets（triangle、path、complete4、two blocks、star、two communities、random seed 7、noisy SBM）只承担 correctness/audit fixture 角色，不与真实数据集结果合并。
