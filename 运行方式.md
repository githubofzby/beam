# 总批处理命令

这里的“总批处理命令”按标题含义应理解为：

- 一次性跑完整实验矩阵
- 覆盖多个 `merge-mode / scoring-backend / candidate-batch-budget`
- 所有结果追加写入**同一个** CSV

当前 V2 之后，建议把总表统一写到一个总 CSV，例如 `./results/beam_all_runs.csv`。

## 总批处理

```bash
THREADS=40
BEAM=./build/beam
DATA=/data/zby/datasets

TOPK=16
ITERS=20
SEED=1
ERROR_BOUND=0.0
GROUP_BATCH_SIZE=64
CUDA_SLICE_MEMORY_MB=0

CSV=./results/beam_all_runs_cu.csv

mkdir -p "$(dirname "$CSV")"
rm -f "$CSV"

export OMP_NUM_THREADS=$THREADS
export OMP_PROC_BIND=close
export OMP_PLACES=cores

for config in \
  "batch-ea|cpu|0" \
  "batch-ea-blocked|cpu|0" \
  "batch-ea-blocked|cuda|0" \
  "batch-ea-blocked|cuda|8192" \
  "batch-ea-blocked|cuda|65536"
do
  IFS="|" read -r mode backend candidate_budget <<< "$config"

  for item in \
    "facebook|$DATA/facebook/facebook.undir.single.txt" \
    "caida|$DATA/caida/caida.undir.single.txt" \
    "email_enron|$DATA/email_enron/email_enron.undir.single.txt" \
    "amazon|$DATA/amazon/amazon.undir.single.txt" \
    "dblp|$DATA/dblp/dblp.undir.single.txt"
  do
    IFS="|" read -r name input <<< "$item"

    echo "===== start name=${name} mode=${mode} backend=${backend} candidate_budget=${candidate_budget} ====="

    cmd=(
      "$BEAM"
      --input "$input"
      --merge-mode "$mode"
      --scoring-backend "$backend"
      --top-k "$TOPK"
      --ea-use-threshold
      --threshold-policy reciprocal
      --iterations "$ITERS"
      --print-offset 1
      --seed "$SEED"
      --error-bound "$ERROR_BOUND"
      --results-csv "$CSV"
    )

    if [ "$mode" = "batch-ea-blocked" ]; then
      cmd+=(--group-batch-size "$GROUP_BATCH_SIZE")
    fi

    if [ "$mode" = "batch-ea-blocked" ] && [ "$backend" = "cuda" ]; then
      cmd+=(
        --candidate-batch-budget "$candidate_budget"
        --cuda-slice-memory-mb "$CUDA_SLICE_MEMORY_MB"
      )
    fi

    "${cmd[@]}"

    status=$?
    echo "===== end name=${name} mode=${mode} backend=${backend} candidate_budget=${candidate_budget} status=${status} ====="

    if [ "$status" -ne 0 ]; then
      exit "$status"
    fi
  done
done
```

# 各 mode 单独批处理命令

这里的“各 mode 单独批处理”按标题含义应理解为：

- 每一类 mode 单独跑
- 每类 mode 写入自己的 CSV
- 便于单独做正确性或性能对比

## `batch-ea + cpu`

```bash
THREADS=40
BEAM=./build/beam
DATA=/data/zby/datasets

TOPK=16
ITERS=20
SEED=1
ERROR_BOUND=0.0

mkdir -p "$(dirname "$CSV")"

export OMP_NUM_THREADS=$THREADS

  "facebook|$DATA/facebook/facebook.undir.single.txt" \
  "caida|$DATA/caida/caida.undir.single.txt" \
  "email_enron|$DATA/email_enron/email_enron.undir.single.txt" \
  "amazon|$DATA/amazon/amazon.undir.single.txt" \
  "dblp|$DATA/dblp/dblp.undir.single.txt"
do
  IFS="|" read -r name input <<< "$item"

  echo "===== start name=${name} mode=batch-ea backend=cpu ====="

  "$BEAM" \
    --input "$input" \
    --merge-mode batch-ea \
    --scoring-backend cpu \
    --top-k "$TOPK" \

    exit "$status"
  fi
done
```

## `batch-ea-blocked + cuda`

当前 V2 下，`batch-ea-blocked + cuda` 最重要的新参数有两个：

- `--candidate-batch-budget`
  现在只控制 CUDA kernel chunk，不再控制高层 slice 切分
- `--cuda-slice-memory-mb`
  控制 blocked CUDA 的 adaptive whole-block slicing 内存预算；`0` 表示自动

### 单个 candidate budget 批处理

BEAM=./build/beam
DATA=/data/zby/datasets

TOPK=16
